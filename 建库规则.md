# Orbitgraph 使用规则 · 自建个人知识库指南

> 这份文档教你**怎么组织自己的 vault**，让 Orbitgraph 的「双环知识图谱」和「AI 批改回路」都能跑起来。
> 只要按这里的字段、目录、命名约定来建库，插件会自动识别，不需要写一行代码。
>
> 不限学科——会计、法考、英语、医学、编程都行。下面所有举例只是示意，换成你自己的内容即可。

---

## 0. Orbitgraph 是什么(一句话)

它把你的笔记画成**两层同心圆**：

- **内环 = 知识点**(概念、考点、原理)——你沉淀的「干货」
- **外环 = 题目/实例**(练习题、真题、案例)——挂在它考查的知识点上

可选的 **AI 批改回路**让你「做题 → 批改 → 掌握度更新 → 图谱自动重新上色」形成闭环。

你需要做的只有一件事:**按规则把笔记写对**。识别、连线、上色、统计全部由插件和批改 Skill 自动完成。

---

## 1. 先决定:你要哪一档?

| 你想要 | 需要满足的规则 | 额外依赖 |
|---|---|---|
| **A. 只看图谱** | 第 3、4 节的「字段规范」（§5 答题纸是批改产物，A 档用不到） | 仅 Obsidian 1.4.0+ |
| **B. 图谱 + 主页仪表盘** | A 的全部 | + Dataview 社区插件 |
| **C. 图谱 + AI 批改闭环** | A 的全部 **+ 第 2 节的目录名必须一字不差** | + Claude 对话插件(如 Claudian)+ Claude Code CLI(仅桌面端) |

> **关键区别**:只做图谱时,目录叫什么名字都行(在插件设置里指给它);一旦要用批改,目录名就是**硬约束**(见下节)。建议新手直接照 C 档的目录名建,以后想开批改就是无缝的。

---

## 2. 目录结构

### 2.1 推荐结构(C 档 · 含批改)

```
你的 vault/
├── 01-知识点库/          # 内环:所有「干货-XXXX.md」放这里(可任意分子文件夹)
│   ├── 第01章/
│   │   └── 干货-0101xxxx.md
│   └── 第02章/
│       └── 干货-0205xxxx.md
├── 02-题库/              # 外环:所有题目 .md 放这里(子目录随意,如 练习题/真题/年份)
│   ├── 练习题/第05章/
│   │   └── P-CH05-007-多选-xxx.md
│   └── 真题/2022/
│       └── Q2022-S2-J09-xxx.md
├── 03-答题纸/            # ⚠️ 不用手建、不要手改 —— 批改时 Skill 自动生成
├── data/
│   └── inbox/           # ⚠️ 答题后插件自动写入的待批改信号,别动
├── 主页.md              # 插件首次启动自动生成的仪表盘
└── .claude/skills/批改/SKILL.md   # 插件首次启动自动释放的批改 Skill
```

### 2.2 硬约束(开了批改就**绝对不能改名**)

批改 Skill 内部把这几个路径写死了。改名 = 批改直接断链。

| 路径 | 谁用 | 改名后果 |
|---|---|---|
| `01-知识点库/` | 内环扫描 + 干货定位 | 掌握度无法回写,热力图不更新 |
| `02-题库/` | 外环扫描 + 题目定位 | 找不到题,批改报错 |
| `03-答题纸/` | 批改结果存放 | 答题纸写到错误位置,分数读不出 |
| `data/inbox/` | 答题信号队列 | 提交的答案永远不被批改 |

### 2.3 软约束(只做图谱时可自定义)

如果你**不用批改、只看图谱**,内/外环目录叫什么都行——在
*设置 → Orbitgraph* 里把「主题(内环)根目录」「实例(外环)根目录」指到你自己的文件夹即可。

## 3. 内环文件:知识点(干货)规范

放在 `01-知识点库/` 下。一个知识点 = 一个 `.md`。

### 3.1 文件命名(开批改时为硬约束)

- 文件名必须是 **`干货-XXXX.md`** 的形式(`XXXX` 是你自定义的编号/短名,如 `干货-0501投资性房地产的范围`)。
- 题目通过 **`[[干货-XXXX]]`** 这种 wikilink 引用它。
- **不要**改成 `kp-XXXX.md`、`知识-XXXX.md` 等——题目里的所有 `[[干货-XXXX]]` 会全部失效,掌握度聚合崩溃。

> 只做图谱、不开批改时,内环文件名可以随意——图谱靠 frontmatter 和链接识别,不强制 `干货-` 前缀。但**建议统一用 `干货-` 前缀**,方便以后开批改。

### 3.2 frontmatter 字段

```yaml
---
概念: 投资性房地产的范围        # 图谱节点显示的标签(没有则用文件名)
tags: [重点]                    # 决定内环分层着色(见下表),可选
前置概念: ["[[干货-0500会计要素]]"]   # → 画一条「前置」连线,可选
包含关系: ["[[干货-0502后续计量]]"]   # → 画一条「包含」连线,可选
易混淆: ["[[干货-0301固定资产]]"]     # → 画一条「易混淆」连线,可选
---
```

**`tags` 与内环分层**(影响节点在哪一层、什么颜色):

| tag 含义 | 层级 |
|---|---|
| 含「总览」或「核心」 | 第 0 层(最内,核心总览) |
| 含「原则」/ principle | 第 1 层 |
| 含「重点」/ key | 第 2 层 |
| 其他 | 第 3 层(默认) |

> 概念字段的别名也认:`概念名`。前置概念也认 `前置`/`先修`/`prereq`;包含关系也认 `包含`/`子概念`/`子主题`。正文里写的 `[[其他干货]]` 也会自动建「包含」连线。

### 3.3 批改自动写入的字段(**别手填、别手改**)

开批改后,Skill 会在干货 frontmatter 里维护这几项,代表这个知识点的掌握程度:

```yaml
考点EMA: 85.2        # 指数滑动平均掌握分(0-100)
掌握状态: 已掌握      # 未学 / 未掌握 / 部分掌握 / 已掌握
答题次数: 6
答对次数: 5
正确率: 0.83
最近答题: 2026-06-15T...
```

主页热力图默认就是按 `考点EMA` 上色的。**手动改这些值会在下次批改时被覆盖**,没有意义。

## 4. 外环文件:题目规范

放在 `02-题库/` 下。一道题 = 一个 `.md`。子目录怎么分都行(按章节、按年份、按来源)。

### 4.1 frontmatter 必需字段(批改依赖)

```yaml
---
题目ID: P-CH05-007                       # 题目唯一标识
正确答案: "ACD"                            # 机械判分依据(见下方格式)
题型: 多选                                 # 单选/多选/判断/计算/综合/简答
考点: ["[[干货-0501投资性房地产的范围]]"]   # ⭐ wikilink 数组,题挂到哪些知识点
核心考点: "[[干货-0501投资性房地产的范围]]" # 数组第一个,弱依赖
章节: "第05章-投资性房地产"
来源类型: 练习题                           # 练习题 / 真题 / 模拟 ...
---
```

| 字段 | 作用 | 缺了会怎样 |
|---|---|---|
| `题目ID` | 答题纸引用 | 答题纸链接失效 |
| `正确答案` | 客观题自动判分 | 答案维度算不出 |
| `题型` | 评分分流 | 走慢通道,不会挂但变慢 |
| `考点` | **决定外环连到内环哪个节点**,也是掌握度聚合的根 | 题成「孤儿节点」,掌握度无法回写 |
| `核心考点` | 冗余引用 | 仅告警,不挂 |
| `章节` / `来源类型` | 写入答题纸做记录 | 答题纸缺字段 |

**`正确答案` 的写法:**

| 题型 | 写法举例 |
|---|---|
| 单选 | `"B"` |
| 多选 | `"ACD"`(连写,大小写不限,顺序无所谓) |
| 判断 | `"√"` 或 `"×"`(也认 对/错、正确/错误) |
| 计算/综合/简答 | 由 AI 按 `## 解析` 的要点命中率打分,`正确答案` 可留参考文本 |

### 4.2 正文必需的二级标题

批改时 Skill 会抓这几个 `##` 段落,做成答题纸的题面快照和评分参考:

```markdown
## 题干
下列各项中,属于投资性房地产的有( )。

## 选项
A. 已出租的建筑物
B. 自用的办公楼
C. 持有并准备增值后转让的土地使用权
D. 已出租的土地使用权

## 答案
ACD

## 解析
B 为自用,属于固定资产……(这段是 AI 评分的主要参照,写充实点)
```

> 标题名要**一字不差**:`## 题干`、`## 选项`、`## 答案`、`## 解析`。改成「## 题目」「## 选择项」之类,题面快照就会缺内容。

### 4.3 批改自动写入的字段(**别手改**)

```yaml
答题次数: 3
答对次数: 1
正确率: 0.33
最近是否正确: false
最近答题: 2026-06-15T...
节点状态: 答错过        # 未做 / 全对 / 已答过 / 答错过
```

外环节点的颜色就来自 `节点状态`(或 `正确率`)。这些由批改维护,手填会被覆盖。

## 5. 答题纸(`03-答题纸/`)——只读,别碰

批改完成后,Skill 会按 **「题库路径去掉第一段,前缀换成 `03-答题纸/`」** 的规则,为每道做过的题生成一份答题纸:

```
02-题库/真题/2022/Q2022-S2-J09.md
        ↓
03-答题纸/真题/2022/Q2022-S2-J09.md
```

里面记录每一遍的作答、四维得分(答案/过程/概念/陷阱)、AI 批示、考点命中情况。
**这是批改的产出物,不要手动新建、改名或编辑**——它既是你的复盘材料,也是掌握度聚合的数据源。

## 6. AI 批改回路:怎么用

### 6.1 前置条件(C 档)

1. 在系统里装好 **Claude Code CLI**,确认终端里 `claude --version` 能跑通(已加入 PATH)。
2. vault 里装一个 Claude 对话插件(如 **Claudian**)。
3. 确认 `.claude/skills/批改/SKILL.md` 存在(插件首次启动自动释放;若缺失,用命令面板 `Orbitgraph: 更新批改 Skill` 重新写出)。
4. 目录名严格按第 2.2 节。

### 6.2 完整闭环

```
①在图谱里点题目 → 答题面板作答 → 提交
        ↓ 插件把作答写成 JSON 放进 data/inbox/
②打开 Claude 对话面板,输入  /批改
        ↓ Skill 扫描 inbox,逐题批改
③Skill 生成/更新答题纸、回写题目 frontmatter、回写干货掌握度
        ↓
④图谱与主页热力图在下次刷新时按新掌握度重新上色
```

> 批改是**手动触发**的:答完题后去 Claude 面板敲 `/批改`。(早期「提交即自动跑 CLI」在 Obsidian 子进程环境里不稳定,目前默认关闭。)

### 6.3 客观题快速通道

单选/多选/判断,如果你**没填思路**,Skill 直接机械判分(秒出),并把总分封顶 80 分——防止纯蒙刷成「已掌握」。想拿满分、想要过程/概念维度的反馈,就在作答时写上你的思路。

---

## 7. 🚫 红线清单:这些绝对不要改

开了批改后,下面任何一项被改动都会让闭环断裂或数据出错:

- ❌ 三个根目录名:`01-知识点库/`、`02-题库/`、`03-答题纸/`
- ❌ 队列目录:`data/inbox/`
- ❌ 干货文件的 `干货-XXXX` 命名前缀
- ❌ 题目 frontmatter 七字段名:`题目ID` / `正确答案` / `题型` / `考点` / `核心考点` / `章节` / `来源类型`
- ❌ 题目正文四个二级标题:`## 题干` / `## 选项` / `## 答案` / `## 解析`
- ❌ 手动编辑由批改维护的字段(干货的 `考点EMA`/`掌握状态`,题目的 `节点状态`/`正确率` 等)和 `03-答题纸/` 里的文件

可以自由做的:在 `01-知识点库/`、`02-题库/` 下任意分子目录;给干货/题目加你自己的其他 frontmatter 字段和正文;调整图谱的力度、半径、颜色、筛选(都在插件设置里,纯展示,不影响数据)。

## 8. 自查清单(建库后对照一遍)

- [ ] 内环文件都在 `01-知识点库/` 下,文件名是 `干货-XXXX.md`
- [ ] 题目都在 `02-题库/` 下,frontmatter 七字段齐全
- [ ] 题目的 `考点:` 是 `["[[干货-XXXX]]"]` 数组,且指向的干货文件**真实存在**(拼错就连不上)
- [ ] 题目正文有 `## 题干 / ## 选项 / ## 答案 / ## 解析`
- [ ] (开批改)`claude --version` 能跑;`.claude/skills/批改/SKILL.md` 存在
- [ ] (主页)装了 Dataview 社区插件

## 9. 排错速查

| 症状 | 多半是因为 |
|---|---|
| 题在图谱里是「孤儿」,不连到任何知识点 | `考点:` 写法不对,或指向的 `干货-XXXX` 文件名拼错/不存在 |
| 点提交后 `/批改` 说收件箱为空 | `data/inbox/` 不存在或被改名 |
| `/批改` 回「我不认识 /批改」 | `.claude/skills/批改/SKILL.md` 缺失或 frontmatter `name: 批改` 被改 |
| 批改报「找不到 `正确答案`」 | 题目 frontmatter 字段被改名 |
| 答题纸题面缺题干 | 正文二级标题不是 `## 题干` 等标准名 |
| 批改「成功」但分数全 0 / 掌握度不动 | 八成是手改过 `03-答题纸/` 文件或自动字段,导致格式被破坏 |
| 主页热力图不更新 | 热力图字段(默认 `考点EMA`)在干货里没有,或还没批改过 |

---

> 想了解每个字段背后的评分公式、EMA 算法、四维评分细则,见 `.claude/skills/批改/SKILL.md`——那是批改逻辑的权威实现。
